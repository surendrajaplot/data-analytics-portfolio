# Operational Analytics and Metric Spike Investigation

## Abstract

This project presents a comprehensive analytical investigation into operational workflows and user engagement metrics using SQL. The objective was to identify operational bottlenecks, evaluate workforce productivity, analyze customer engagement behaviour, and investigate metric fluctuations that may influence business performance.

Advanced SQL techniques including Common Table Expressions (CTEs), Window Functions, Cohort Analysis, and Rolling Aggregations were applied to transform raw data into actionable business insights.

---

## Introduction

Modern organisations generate significant volumes of operational and customer interaction data. Extracting value from this data requires analytical techniques capable of identifying trends, anomalies, and performance indicators.

This project focuses on:

* Operational Analytics
* User Engagement Analytics
* Retention Analysis
* Performance Monitoring

The primary objective was to support evidence-based decision making through data-driven insights.

---

## Research Objectives

* Evaluate operational efficiency through job review analysis.
* Measure throughput and operational performance.
* Analyse language distribution patterns.
* Identify duplicate records affecting data quality.
* Investigate user growth trends.
* Perform cohort-based retention analysis.
* Assess device-level engagement behaviour.
* Evaluate email campaign effectiveness.

---

## Methodology

### Data Collection

Data was imported into MySQL Workbench for analytical processing.

### Data Preparation

Datasets were examined for missing values, duplicate records, and inconsistencies.

### Exploratory Data Analysis

SQL queries were developed to generate key operational and engagement metrics.

### Insight Generation

Business-focused interpretations were derived from analytical findings.

---

## Operational Analytics

### Job Review Analysis

Analysed hourly review volumes to identify productivity trends and workload distribution.

### Throughput Analysis

Calculated rolling throughput metrics to evaluate operational performance over time.

### Language Distribution

Investigated language usage trends to support workforce planning and localisation strategies.

### Data Quality Assessment

Performed duplicate record detection to improve dataset integrity.

---

## User Engagement Analytics

### User Growth Analysis

Examined registration patterns and growth trends.

### Weekly Engagement Analysis

Measured active user participation over time.

### Cohort Retention Analysis

Evaluated user retention behaviour across multiple registration cohorts.

### Device-Level Engagement

Compared user interaction patterns across device categories.

### Email Campaign Performance

Assessed open rates and click-through rates to evaluate communication effectiveness.

---

## Key Findings

* Rolling throughput metrics provided a more stable measure of operational performance.
* User retention declined significantly after initial engagement.
* Device-specific behaviour patterns influenced engagement levels.
* Language distribution revealed resource allocation opportunities.
* Email campaigns achieved acceptable open rates but lower click-through performance.

---

## Business Recommendations

1. Strengthen user onboarding strategies.
2. Adopt rolling throughput as a primary KPI.
3. Conduct routine data quality audits.
4. Implement A/B testing for email campaigns.
5. Improve device-specific user experiences.
6. Optimise workforce allocation based on language demand.

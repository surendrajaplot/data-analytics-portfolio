# Operational Analytics and Metric Spike Investigation

## Project Overview

This project focuses on Operational Analytics and Metric Spike Investigation using SQL. The objective was to analyze operational performance, user engagement, retention patterns, and marketing effectiveness to support data-driven business decisions.

## Business Problem

Organizations generate large volumes of operational and user activity data. The challenge is to transform this raw information into actionable insights that can improve operational efficiency, customer engagement, and strategic planning.

## Tools and Technologies

* SQL
* MySQL Workbench
* Microsoft Excel
* Google Sheets

## Case Study 1: Job Data Analysis

### Key Analyses

* Jobs Reviewed Per Hour
* 7-Day Rolling Throughput
* Language Distribution Analysis
* Duplicate Record Detection

### Key Insights

* Peak review periods were identified through hourly workload analysis.
* Rolling throughput metrics provided a more reliable measure of performance than daily throughput.
* Language distribution analysis supported workforce planning decisions.
* Duplicate record detection improved overall data quality.

## Case Study 2: Metric Spike Investigation

### Key Analyses

* Weekly User Engagement
* User Growth Trends
* Cohort Retention Analysis
* Device-Based Engagement Analysis
* Email Campaign Performance

### Key Insights

* User retention declined significantly after the first week.
* Weekly active user trends revealed engagement fluctuations.
* Device-specific behavior patterns influenced platform usage.
* Email open rates were satisfactory, while click-through rates indicated optimization opportunities.

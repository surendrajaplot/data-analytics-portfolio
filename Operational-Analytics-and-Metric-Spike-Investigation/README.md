# Operational Analytics and Metric Spike Investigation

## Project Overview

This project investigates operational performance metrics and user engagement behaviour using SQL-based analytics. The objective was to identify workflow inefficiencies, evaluate business performance indicators, and generate data-driven recommendations.

## Business Problem

Large organizations generate substantial operational and customer interaction data. This project demonstrates how SQL analytics can transform raw data into actionable business insights.

## Tools & Technologies

- SQL
- MySQL Workbench
- Microsoft Excel
- Google Sheets

## Case Study 1: Job Data Analysis

### Analysis Performed

- Jobs Reviewed Per Hour
- 7-Day Rolling Throughput
- Language Distribution Analysis
- Duplicate Record Detection

### Key Findings

- Peak workload periods were identified through hourly review analysis.
- Rolling throughput metrics provided more reliable performance monitoring.
- Language distribution analysis supported workforce planning.
- Duplicate detection improved data quality assessment.

## Case Study 2: Metric Spike Investigation

### Analysis Performed

- Weekly User Engagement
- User Growth Analysis
- Cohort Retention Analysis
- Device-Level Engagement Analysis
- Email Engagement Analysis

### Key Findings

- User retention declined significantly after Week 1.
- Weekly engagement trends revealed fluctuations in platform activity.
- Device-specific behavioural differences were identified.
- Email click-through rates highlighted optimisation opportunities.
